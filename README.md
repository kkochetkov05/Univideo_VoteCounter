# Скрипт для подсчета голосов на конкурсе "Унивидение"

### Быстрый старт:
#### Пользователь:
+ Скачать `/dist/UnivideoVoteCounter.exe`
+ Создать отдельную папку под приложение (желательно)
+ Запустить `UnivideoVoteCounter.exe`

Первый запуск будет долгим.
В директории с программой создатся `config.json`.
После запуска интерфейса укажите пути на все файлы через `Обзор`, укажите дату, за которую нужно считать голоса, сохраните конфигурацию, и можно запускать подсчет.

*Подсказка:* самая частая ошибка `permission denied` - закройте файл, в который должны загружаться результаты.
#### Разработчик:
+ `git clone https://github.com/kkochetkov05/Univideo_VoteCounter`
+ Создать виртуальное окружение (желательно)
+ `pip install -r requirements.txt`
+ `python main.py`

### Главные файлы

- **main.py**  
  Точка входа приложения. Создаёт окно Tkinter и запускает `VoteCounterGUI`.

- **gui.py**  
  Графический интерфейс:
  - ввод путей к Excel‑файлам со студентами и голосами;
  - выбор даты голосования;
  - сохранение/загрузка `config.json`;
  - запуск пайплайна обработки (`pipeline.start()`).

- **pipeline.py**  
  Основной пайплайн:
  - читает конфигурацию `config.json`;
  - загружает Excel‑файлы через `source.data.input`;
  - форматирует таблицы (STs и голоса);
  - фильтрует голоса по дате;
  - считает рейтинги факультетов (`build_result_table`);
  - сохраняет результат в Excel через `source.data.output`.

- **config.py**  
  Определяет путь к `config.json` с учётом запуска из исходников или из `.exe`.

- **config.json**  
  Настройки проекта: пути к Excel‑файлам, путь вывода результата и дата голосования.

### Модуль обработки голосов

- **source/core/name_mapping.py**  
  Функция `is_name_in_db(formName)` - сопоставляет название факультета из формы с названиями направлений в таблицах студентов.

- **source/core/vote_filter.py**  
  - `_normalize_fio` - нормализация ФИО (убирает ё/е, пробелы).  
  - `_get_student_record` - поиск студента по st‑логину в объединённой таблице STs.  
  - `_get_faculty_choices` - извлекает факультеты, за которые человек проголосовал.  
  - `process_vote_row` - валидация одного голоса (существование студента, совпадение ФИО, ≤3 факультетов).  
    - для студентов: проверка существования в STs, совпадения ФИО, отсутствия своего факультета в выборе, не более 3 факультетов; 
    - для сотрудников: проверка логина формата st######, не более 3 факультетов.
  - `build_result_row` - формирует одну строку результата: chat_id, От кого, 1, 2, 3, mail (для сотрудников в «От кого» стоит Сотрудник, для студентов - направление).
  - `build_result_table` - по всем голосам строит итоговую таблицу формата result_1.xlsx, учитывая только первый валидный голос каждого st.
### Ввод/вывод данных

- **source/data/input.py**  
  - `load_config` - чтение `config.json`.  
  - `load_requied_files` - загрузка Excel‑файлов студентов и голосов.  
  - `formate_tables` - приведение таблиц к единому формату (столбцы `st`, `ФИО`, направления, факультеты).

- **source/data/output.py**  
  - `export_results_to_excel` - сохранение ранжированного списка факультетов в Excel‑файл.
